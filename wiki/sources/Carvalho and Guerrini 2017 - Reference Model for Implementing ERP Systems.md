# Carvalho and Guerrini 2017 - Reference Model for Implementing ERP Systems

## Summary
- Proposes a reference model for ERP implementation as an innovation-agent network involving customer, supplier, and consultancy.
- Uses multi-case study data from seven companies and the Enterprise Knowledge Development (EKD) modelling method.
- Argues ERP implementation requires coordination of goals, business rules, processes, actors/resources, concepts, technical components, and requirements.
- Finds analyzed implementations were weakly systematized: average global deployment score was 63.6/177, and only one company exceeded 50%.
- Highlights over-transfer of responsibility to consultants and lack of systematic cooperation/network competence.

## Details
The model moves ERP implementation from consultancy-led software configuration toward internal capability building and knowledge internalization. The proposed process includes forming an internal implementation network, modelling company/process integration needs, reconfiguring the network with external consultants, implementing ERP, internalizing knowledge before dissolving the network, and maintaining the system through innovation management and continuous improvement.

# Shen et al. 2010 - Systems Integration and Collaboration in AEC FM

## Summary
- Comprehensive review of systems integration and collaboration technologies in architecture, engineering, construction, and facilities management.
- Defines interoperability as the ability of diverse software/hardware systems to manage and communicate product/project data smoothly.
- Distinguishes data interoperability via models/standards from framework interoperability via protocols/languages.
- Reviews approaches including web-based systems, distributed objects/components, software agents, web services, semantic web, RFID/wireless sensors, collaborative virtual environments, and virtual organizations.
- Notes AEC/FM interoperability is hindered by fragmentation, many small firms, heterogeneous tools, lifecycle complexity, and competing standards.

## Details
The paper uses AEC/FM as a domain where integration failure has measurable cost and lifecycle consequences. A common neutral data model can reduce repeated data recreation and inconsistencies, but standards can have competing interpretations. Loosely coupled integration via web services, agents, and semantic technologies is useful where legacy applications and distributed stakeholders cannot share one common database.

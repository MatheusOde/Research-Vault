# The Role Of A Protocol In A Systematic Literature Review

## Summary
- Tutorial paper arguing that literature review protocols should cover the end-to-end SLR journey, not only coding.
- Defines a protocol as a predetermined, detailed plan and living document for the review.
- Protocol components include goals, topic areas/phenomena, search strategy, paper screening, extraction summary, tool use, coding plans/rules/codebooks, reliability/corroboration, and reporting plans.
- Frames protocols as tools for quality assurance, collaboration, transparency, efficiency, and reproducibility.

## Details
The paper positions the literature review protocol as a blueprint and living document. It should articulate review context and purpose, research questions, search strategy, paper selection, coding/analysis, planned reporting, and quality assurance.

The authors argue that protocols reduce researcher bias, improve credibility, enable replication, help teams coordinate, and support novices. They also note that many literature reviews either omit protocols or mention them narrowly, usually only for coding.

The paper gives detailed guidance for protocol elements. Researchers should define goals, identify key phenomena and synonyms, decide where to search and which fields/sources to use, justify initial criteria, build database-specific search strings, document screening heuristics, summarize paper extraction, select and configure tools, define coding cycles, coding rules, codebooks, coder corroboration/intercoder reliability, and sketch reporting outputs in advance.
